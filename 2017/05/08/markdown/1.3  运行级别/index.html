运行级别介绍

安装虚拟机的时候选择了最小化安装（minimal），默认的运行级别是 3

查看运行级别

runlevel

[root@localhost ~]# runlevel
N 3

查看所有级别

cat /etc/inittab

Default runlevel. The runlevels used are:
0 -- halt (Do NOT set initdefault to this)  #关机
1 -- Single user mode    #单用户模式
2 -- Multiuser, without NFS (The same as 3, if you do not have networking)  #多用户模式
3 -- Full multiuser mode  #完全版多用户模式
4 -- unused     #空余出来的，不用的
5 -- X11        #图形界面模式
6 -- reboot (Do NOT set     # 重开机

重开机命令

init 6

reboot

shutdown -r now

关机

init 0

shutdown -h now

单用户模式

类似于windows系统的安全模式，这个模式只启用关键应用，一般用来更改root用户密码。但是前提没有设置grup密码。

进入方法：

  • 启动服务器时，出现提示在3秒按下任意一个键
  • 进入选择界面，按e进入编辑模式
  • 选择第二行，内核信息，再按e进行编辑
  • 在最后的位置加入 1 或者 s 或者 s 或者single
  • 回车确定，然后按b（boot）系统就会启动，进入单用户模式

修改root密码

password

修改成功后，重启服务器就可以使用新的密码登录，

救援模式

类似于windows系统的pe系统，他们都是属于内存操作系统，在内存中运行。一般用于Linux系统无法正常启动时修复系统，修改root密码（设置grup密码）等。

进入方法：

  • 进入bios，选择CD-ROM为第一启动项
  • 开机后，选择第三项（rescue）
  • 进入界面后进行选择操作
  • 语言——选择英语（命令行模式，中文无法支持）
  • 键盘——默认
  • 网络——不需要开启
  • 完成之后系统会被挂载到/mnt/sysimage

进入到原来系统命令

chroot /mnt/sysimage

退出

exit

注意：

  • 进入原来的系统后可以直接使用psssword来修改root密码
  • 重新加载系统要先把启动项改回来
